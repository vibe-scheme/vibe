## Seed compiler v0.0.2

Self-hosted `vibe_kernel` built from this tag’s kernel sources.

### Why this release

The kernel now includes a macro expansion phase (`define-syntax` / `syntax-rules` with flat linear patterns). The previous seed (`v0.0.1-seed`) predates that work and cannot compile the current kernel. This binary is the new bootstrap for clean checkouts.

### What to use it for

- Default bootstrap for `main` (see `build.sh` and `RELEASING.md`).
- Checkpoint before a larger **kernel** migration that relies on macros; keep this tag if you need a compiler that matches pre-migration expectations.

### Asset

- **Name**: `vibe_kernel_seed` (required by `build.sh` and download URLs).
- **Platform**: arm64-apple-darwin (see `RELEASING.md`).

### Historical seed

- `v0.0.1-seed` remains available for bootstrapping older commits that do not require the in-kernel macro expander.

### Publishing

Maintainers: `./build.sh release-seed v0.0.2-seed` (or the **Release seed compiler** GitHub Action). After this release exists, bump the default `SEED_TAG` in `build.sh` and docs when you want clean clones to use it without `VIBE_SEED_TAG`.
