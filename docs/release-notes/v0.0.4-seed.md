## Seed compiler v0.0.4

Self-hosted `vibe_kernel` built from this tag’s kernel sources.

### Why this release

The macro expander now supports **nested list subpatterns** after the keyword (e.g. `(m (a b) c)`) and **final `...`** in patterns and templates: ellipsis-bound pattern variables hold **proper-list** AST values, and templates replicate/splice `(subtemplate ...)` in the usual `syntax-rules` way. A **codegen** fix ensures `llvm:label` bodies skip the quoted label name when evaluating DSL, so the larger expander emits valid LLVM.

The previous default seed (`v0.0.3-seed`) cannot compile a kernel whose macros use **nested patterns** or **`...`**.

### What to use it for

- Default bootstrap for `main` after this tag (see `build.sh` and `RELEASING.md`).
- Baseline when adopting **ellipsis-containing** or **nested-pattern** macros in the compiler and prelude.

### Known gaps (not required for this seed)

- Non-final “split the middle” ellipsis patterns (e.g. `(m x ... y)`) are still unsupported; see `docs/design/macro-system.md`.
- Hygienic `syntax-rules` remains future work.

### Asset

- **Name**: `vibe_kernel_seed` (required by `build.sh` and download URLs).
- **Platform**: arm64-apple-darwin (see `RELEASING.md`).

### Older seeds

- **`v0.0.3-seed`**: literals list, multiple clauses, top-level install-after-expand; **no** nested subpatterns or ellipsis.
- **`v0.0.2-seed`**: linear `syntax-rules` only (no literals list, no install-after-expand as in v0.0.3).
- **`v0.0.1-seed`**: pre–macro-expander chain; use via `VIBE_SEED_TAG` for historical commits.

### Publishing

Maintainers: `./build.sh release-seed v0.0.4-seed` (or the **Release seed compiler** GitHub Action with tag `v0.0.4-seed`). After the release exists, the default `SEED_TAG` in `build.sh` should be `v0.0.4-seed` so clean clones bootstrap without `VIBE_SEED_TAG`.
