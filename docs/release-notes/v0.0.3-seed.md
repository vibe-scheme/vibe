## Seed compiler v0.0.3

Self-hosted `vibe_kernel` built from this tag’s kernel sources.

### Why this release

The macro expander gained **syntax-rules literals**, **multiple clauses** (first match wins), and **top-level install-after-expand**: a form that macro-expands to `(define-syntax …)` is registered like a raw top-level `define-syntax`. That lets preludes use **`define-vibe-syntax`** and similar wrappers without duplicating `define-syntax` at every site.

The previous default seed (`v0.0.2-seed`) cannot compile a kernel that relies on these behaviors.

### What to use it for

- Default bootstrap for `main` after this tag (see `build.sh` and `RELEASING.md`).
- Baseline for the next phase: richer `syntax-rules` (ellipsis, nested patterns) and kernel refactors that depend on the current expander.

### Asset

- **Name**: `vibe_kernel_seed` (required by `build.sh` and download URLs).
- **Platform**: arm64-apple-darwin (see `RELEASING.md`).

### Older seeds

- **`v0.0.2-seed`**: linear `syntax-rules` only (no literals list, single-clause parsing path, no install-after-expand).
- **`v0.0.1-seed`**: pre–macro-expander chain; use via `VIBE_SEED_TAG` for historical commits.

### Publishing

Maintainers: `./build.sh release-seed v0.0.3-seed` (or the **Release seed compiler** GitHub Action with tag `v0.0.3-seed`). After the release exists, the default `SEED_TAG` in `build.sh` should be `v0.0.3-seed` so clean clones bootstrap without `VIBE_SEED_TAG`.
