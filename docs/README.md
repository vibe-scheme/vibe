# Vibe Documentation

This directory contains all documentation for the Vibe language project.

## Structure

### `design/`

Formal design documents and architectural plans:
- `vision.md` — Mission, goals, architecture, downstream possibilities
- `macro-system.md` — Macro implementation roadmap and strategy
- `primitive-forms.md` — Primitive vs derived form analysis
- `r7rs-compliance.md` — R7RS Small compliance tracker
- `cross-compilation-plan.md` — Plan for multi-architecture support
- `bootstrap-plan.md` — Historical bootstrap compiler plan (retired)

### `chats/`

Recorded development conversations, numbered sequentially (e.g., `0050-macro-expander-foundation.md`).

### `pages/`

GitHub Pages site (`index.html`) — public-facing project introduction, status, and links.

### `examples/`

Example programs and tutorials demonstrating Vibe features (to be added).

## Documentation Guidelines

### Design Documents

Design documents should cover:
- Architecture and component design
- Data structures and interfaces
- Implementation plans and phases
- Technical considerations and trade-offs

### Chat Documentation

Each development conversation should be documented with:
- Date and context
- Key decisions made
- Implementation details discussed
- Important notes or considerations
- Links to related documents or code

### Examples

Example programs should:
- Demonstrate specific Vibe features
- Include comments explaining the code
- Show best practices
- Be executable and tested

## Contributing Documentation

When adding new documentation:
1. Place design documents in `design/`
2. Record conversations in `chats/` with sequential numbering
3. Add examples to `examples/` with descriptive names
4. Update this README if adding new categories
