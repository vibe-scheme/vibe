#+TITLE: Vibe Language Module for Doom Emacs
#+AUTHOR: Vibe Project

* Overview

This Doom module provides Vibe language support: ~vibe-mode~ (scheme-derived with custom indentation) and ~vibe-ts-mode~ (tree-sitter integration when the grammar is built).

* Prerequisites

- Emacs 29+ (for built-in treesit)
- Doom Emacs
- Node.js (for ~tree-sitter generate~)
- C compiler (cc, gcc, or clang)

* Installation

Packages (~vibe-mode~, ~vibe-ts~) are pulled from the public repo at
https://github.com/vibe-scheme/vibe. No path configuration or symlinks
for the packages themselves.

** 1. Add the module to Doom

The module must live under ~$DOOMDIR/modules/~. Clone the Vibe repo and
create a symlink:

#+begin_src shell
git clone https://github.com/vibe-scheme/vibe.git ~/Source/vibe
ln -s ~/Source/vibe/doom-modules/lang/vibe ~/.config/doom/modules/lang/vibe
#+end_src

Add ~vibe~ to the ~:lang~ section in ~$DOOMDIR/init.el~:

#+begin_src elisp
(lang
  ...
  vibe
  ...)
#+end_src

** 2. Sync and restart

#+begin_src shell
doom sync
#+end_src

Restart Emacs. ~vibe-mode~ will activate for ~.vibe~ files.

** 3. Tree-sitter (optional)

For ~vibe-ts-mode~ with syntax highlighting, build the grammar:

#+begin_src shell
cd ~/Source/vibe/tree-sitter-vibe
npm install
./build.sh
#+end_src

This produces ~libtree-sitter-vibe.so~ (Linux) or ~libtree-sitter-vibe.dylib~
(macOS). Ensure ~treesit-extra-load-path~ includes the build directory.

* Fallback

If the grammar is not built or fails to load, ~vibe-ts-mode~ still activates but uses ~vibe-mode~ behavior (scheme-mode with custom indent specs for ~llvm:define-function~, etc.). No tree-sitter features will be available.

* Local development

To test changes to ~vibe-mode.el~ or ~vibe-ts.el~, override the recipes in
~$DOOMDIR/packages.el~ (before the ~(doom!)~ call):

#+begin_src elisp
(package! vibe-mode :recipe (:local-repo "~/Source/vibe/emacs"))
(package! vibe-ts :recipe (:local-repo "~/Source/vibe/emacs"))
#+end_src

* Troubleshooting

- **Grammar not loading**: Ensure ~tree-sitter-vibe/build/~ exists and contains ~libtree-sitter-vibe.so~ or ~libtree-sitter-vibe.dylib~. Run ~./build.sh~ in ~tree-sitter-vibe/~.
- **Wrong mode**: Check that ~treesit-extra-load-path~ includes the build directory. Use ~M-x treesit-language-available-p~ with ~vibe~ to verify.
- **Indentation issues**: ~vibe-mode~ uses ~scheme-indent-function~ with indent spec 1 for define-like forms. If tree-sitter is active, ~treesit-simple-indent-rules~ apply.
